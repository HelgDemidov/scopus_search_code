## 1. `/explore` показывает нули — данные из Supabase не отображаются

**Статус:** 🔴 Актуально. Сетевой барьер снят (коммит `bf1b2fa`), но данные по-прежнему пусты.

**Диагноз:** `GET /articles/stats` возвращает нулевые значения. Причина не установлена — нужна диагностика. Три вероятных причины:

1. **Мисматч имён таблиц:** ORM-модели и реальные таблицы в Supabase могут расходиться по именам.
2. **Неверная БД:** `DATABASE_URL` в Railway может указывать не на ту Supabase-БД, куда писал seeder.
3. **Отсутствующие данные:** `GET /articles/stats` может работать правильно, но таблица `articles` пуста.

**План диагностики:**

1. Проверить в **Railway Logs**: отправить `GET /articles/stats` — что возвращает бэкенд, `{"total": 0}` или реальные числа.
2. Проверить в **Railway Dashboard**: `DATABASE_URL` указывает на ту же Supabase-БД, куда писал seeder.
3. Проверить наличие записей в Supabase: открыть Supabase Dashboard → Table Editor → `articles` — есть ли данные.
4. Сопоставить имена таблиц в ORM-моделях (`app/models/`) с реальными именами таблиц в Supabase.

## 2. Структура страниц: отсутствует маршрут `/article/:id`

**Статус:** 🔴 Актуально. Страница деталей статьи (§4.5 ТЗ) не реализована.

**Диагноз:** В `App.tsx` отсутствует маршрут `/article/:id` и соответствующий компонент `ArticlePage`.

**Текущая структура страниц в `App.tsx`:**

| Путь | Компонент | Статус по ТЗ §4 |
|---|---|---|
| `/` | `HomePage` | Реализована (§4.1) |
| `/explore` | `ExplorePage` | Реализована (§4.2), данные пусты |
| `/auth` | `AuthPage` | Реализована (§4.3) |
| `/auth/callback` | `OAuthCallback` | Реализована |
| `/profile` | `ProfilePage` (protected) | Реализована (§4.4) |
| `/article/:id` | **отсутствует** | Не реализована (§4.5 по ТЗ) |

**Необходимо:** Создать компонент `ArticlePage.tsx` и зарегистрировать маршрут в `App.tsx`. Работа запланирована на ветку `database-search-debugging`.
