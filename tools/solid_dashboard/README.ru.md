# SOLID Verifier Dashboard

Config‑ориентированный CLI‑инструмент для анализа Python‑проектов на соответствие принципам SOLID и слоистой архитектуре. Объединяет несколько статических анализаторов в единый пайплайн и выдаёт машиночитаемый отчет.

> **Статус:** Сейчас инструмент живёт внутри репозитория `Scopus Search Code` в каталоге `tools/solid_dashboard/`. В дальнейшем он будет вынесен в отдельный репозиторий и оформлен как самостоятельный Python‑пакет.[file:435]

---

## Возможности

### Реализовано

- **Config‑driven архитектура**
  - Единый файл `solid_config.json` описывает корень пакета, логические слои и игнорируемые директории.[file:435]
  - Позволяет использовать верификатор повторно в разных Python‑проектах.

- **Анализ цикломатической сложности (Radon)**
  - Считает сложность функций/методов и присваивает ранги (A–F).[file:435]
  - Помогает находить потенциальные нарушения SRP (слишком “толстые” методы).[file:504]

- **Анализ связности классов (cohesion)**
  - Вычисляет метрику связности (LCOM‑подобные показатели) для классов.[file:435]
  - Подсвечивает классы с низкой связностью, которые совмещают несколько обязанностей.

- **Адаптер AST‑графа импортов**
  - Строит граф импортов между модулями на основе AST.[file:504]
  - Относит каждый модуль к слою (routers, services, infrastructure, models, interfaces, other) и выдает список внутренних зависимостей между слоями.

- **Архитектурные контракты через import‑linter**
  - Использует Python API `import-linter` и конфиг `.importlinter` в корне хост‑проекта.[file:435]
  - Применяет правила слоистой архитектуры (например, routers → services → infrastructure → models; запрет обратных импортов).[file:504]

- **Анализ графа вызовов (Pyan3)**
  - Генерирует статический call‑graph, удаляет дубликаты ребер и умеет фильтровать эндпоинты FastAPI‑роутеров.[file:435]
  - Даёт основу для поиска мёртвого кода и более глубокого анализа LSP/OCP.[file:504]

### Запланировано

- **LLM‑адаптер (AI‑анализ)**
  - Опциональные режимы `--ai` / `--ai-strict` с использованием OpenAI/Ollama‑совместимых API.[file:435]
  - Heuristic‑анализ нарушений OCP, LSP, ISP и DIP, которые трудно отловить чисто статическими средствами.

- **HTML‑дашборд**
  - HTML‑отчёт на базе Jinja2 с графами на vis.js и интерфейсом на Bootstrap.[file:435]
  - Визуализирует сложность, связность, граф импортов, граф вызовов и AI‑находки с фильтрами и drill‑down.

- **Сравнение с baseline**
  - Возможность сохранять JSON‑baseline и сравнивать новые запуски с эталоном.[file:435]
  - Подсвечивает архитектурные регрессии между коммитами/ветками.

## Структура проекта

Верификатор SOLID сейчас встроен в репозиторий `scopus_search_code` как отдельный инструмент. Ниже показаны только файлы, относящиеся к нему.

```text
scopus_search_code/
├── .importlinter                     # Контракты слоистой архитектуры хост-приложения (используются ImportLinterAdapter)
├── run_solid_dashboard.py            # Удобная точка входа для запуска SOLID-пайплайна
└── tools/
    └── solid_dashboard/              # Корень инструмента SOLID Verifier
        ├── solid_dashboard/          # Основной Python-пакет
        │   ├── adapters/             # Обертки над внешними анализаторами
        │   │   ├── cohesion_adapter.py
        │   │   ├── import_graph_adapter.py
        │   │   ├── import_linter_adapter.py
        │   │   ├── pyan3_adapter.py
        │   │   └── radon_adapter.py
        │   ├── interfaces/           # Внутренние абстракции
        │   │   └── analyzer.py       # Протокол IAnalyzer (name + run(target_dir, context, config))
        │   ├── config.py             # Загрузка и валидация solid_config.json
        │   ├── pipeline.py           # Последовательный пайплайн и распространение контекста
        │   ├── schema.py             # Типы конфигурации/результатов (Pydantic / TypedDict)
        │   └── __main__.py           # Внутренний CLI (аргументы, печать отчета)
        ├── tests/                    # Тесты самого верификатора
        │   └── sample_project/       # Минимальный проект-фикстура для адаптеров
        ├── report/                   # Генерируемые отчеты
        │   └── solid_report.log      # Последний JSON-отчет пайплайна
        ├── solid_config.json         # Маппинг слоёв и правила игнора для конкретного проекта
        └── requirements.txt          # Зависимости верификатора (radon, pyan3, import-linter и др.)
```

## Заметки

- `run_solid_dashboard.py` лежит в корне хост-проекта и намеренно остается тонкой оберткой:
  он только вычисляет пути и вызывает `python -m tools.solid_dashboard.solid_dashboard ...`.
- `run_tests.py` и `run_export_skeleton.py` относятся к основному сервису (тесты и LLM-контекст)
  и сознательно не входят в публичную поверхность SOLID-верификатора.[file:504]

## Использование

Запускать инструмент нужно из корня хост-проекта (`scopus_search_code`):

```bash
python run_solid_dashboard.py
```
Этот скрипт:

- Находит целевой каталог (сейчас ./app) и конфиг (./tools/solid_dashboard/solid_config.json).[file:504]

- Вызывает внутренний CLI:

```bash
python -m tools.solid_dashboard.solid_dashboard \
    --target-dir ./app \
    --config ./tools/solid_dashboard/solid_config.json
```
- Последовательно запускает все настроенные адаптеры (Radon, Cohesion, Import Graph, Import Linter, Pyan3).[file:504]

- Печатает JSON-результат в stdout и записывает его в файл:
```text
tools/solid_dashboard/solid_dashboard/report/solid_report.log
```

Вы можете закоммитить solid_config.json и .importlinter в любой другой Python-проект, скопировать туда каталог tools/solid_dashboard/ и сохранить тот же контракт запуска. После выделения в отдельный пакет точка входа превратится в глобальную CLI-команду (например, solid-dashboard) с теми же флагами и поведением.[file:435]
